The _Voucher Return Requests_ page allows admins to review and take action on employee requests to return unused or partially used vouchers.

##### **Where to Find It**

From the left menu, click **Vouchers → Return Requests** to open your approval dashboard.

![[nexpense_vouchers_return_requests_v1.png]]

##### **1. Search Requests**

Use the search bar to quickly locate specific requests by:

- Employee name
    
- Voucher code
    
- Reason keywords
    

This helps you filter entries without scrolling through the entire list.

##### **2. Status Filter**

Use the dropdown filter to view requests based on their current status:

- **Pending** – Requests waiting for admin review
    
- **Approved** – Requests that have been approved
    
- **Rejected** – Requests that were declined
    

This makes it easy to focus on items that need your attention.

##### **3. Refresh Button**

Click **Refresh** to update the list with the most recent requests submitted or modified by employees.

#### **Table Columns**

Each return request (when available) will display the following information:

- **Voucher** – The voucher associated with the return request
    
- **Employee** – Who submitted the request
    
- **Return Amount** – The amount the employee wants to return
    
- **Reason** – The reason provided by the employee
    
- **Status** – Pending / Approved / Rejected
    
- **Requested** – Date and time when the request was made
    
- **Actions** – Options for reviewing and responding to the request
    

#### **Empty State**

If no requests match your filters or no requests have been made, you will see:

> **“No return requests found”**

![[nexpense_vouchers_manage_vouchers_return_requests_dashboard_v1.png]]
#### **Status Options**

- **All Status**  
    Shows every return request, regardless of its state.
    
- **Pending**  
    Displays requests that are waiting for review or admin action.
    
- **Approved**  
    Shows requests that have already been reviewed and approved.
    
- **Rejected**  
    Lists requests that were either invalid or declined.
    

#### **How to Use**

1. Click on the **Status** dropdown.
    
2. Select the desired filter option.
    
3. The table below will refresh automatically based on your selection.
    

This makes it easy to manage return requests efficiently and stay on top of pending actions.
